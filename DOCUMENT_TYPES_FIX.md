# 🔧 תיקון: הבהרת סוגי מסמכים - גרסה 1.0.2

## ❌ מה הייתה הבעיה?

כשניסית ליצור **בקשת תשלום (Payment Request)**, המערכת יצרה **תעודת החזרה (Return/Credit Note)** במקום.

### הסיבה:

הסוגים היו מוצגים רק באנגלית והשמות היו דומים:
- ❌ "Return/Credit" (ערך 9)
- ❌ "Payment Request" (ערך 10)

**קל מאוד לבלבל ביניהם!**

---

## ✅ מה תיקנו?

הוספנו **תרגום עברי** לכל סוגי המסמכים כדי למנוע בלבול:

### לפני (v1.0.1):
```
Return/Credit          ← לא ברור מה זה
Payment Request        ← לא ברור מה זה
```

### אחרי (v1.0.2):
```
Return/Credit Note (תעודת החזרה)     ← ברור שזה החזרה!
Payment Request (בקשת תשלום)          ← ברור שזו בקשת תשלום!
```

---

## 📋 רשימה מלאה של סוגי המסמכים

עכשיו כל הסוגים מוצגים כך:

| ערך | שם באנגלית | שם בעברית | מתי להשתמש |
|-----|-----------|-----------|-----------|
| **0** | Invoice | חשבונית | חשבונית רגילה |
| **1** | Tax Invoice | חשבונית מס | **הכי נפוץ** - למכירות רגילות |
| **2** | Receipt | קבלה | לתקבולים בלבד |
| **3** | Tax Invoice/Receipt | חשבונית מס/קבלה | מכירה + תשלום מיידי |
| **4** | Credit Invoice | חשבונית זיכוי | זיכוי ללקוח |
| **5** | Proforma Invoice | חשבונית עסקה | חשבונית ללא מע"מ |
| **6** | Delivery Note | תעודת משלוח | תעודת משלוח |
| **7** | Quote | הצעת מחיר | הצעות מחיר |
| **8** | Order | הזמנה | הזמנות |
| **9** | Return/Credit Note | תעודת החזרה | **החזר מוצר** |
| **10** | Payment Request | בקשת תשלום | **בקשה לתשלום** ✅ |

## 💡 טיפים נוספים

### בקשת תשלום vs תעודת החזרה:

**בקשת תשלום (Payment Request):**
- 💰 משמש לבקש תשלום מלקוח
- 📧 ניתן לשלוח במייל עם קישור לתשלום
- ✅ משמש כאשר אתה **רוצה לקבל כסף**

**תעודת החזרה (Return/Credit Note):**
- 📦 משמש כאשר לקוח מחזיר מוצר
- 💸 זיכוי ללקוח
- ⬅️ משמש כאשר אתה **מחזיר כסף**

**עכשיו קל לזהות!** 🎯
