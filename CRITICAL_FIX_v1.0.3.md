# 🚨 תיקון קריטי - גרסה 1.0.3

## ❌ הבעיה שתוקנה

כאשר בחרת **"Payment Request (בקשת תשלום)"**, המערכת יצרה **"Return/Credit Note (תעודת החזרה)"** במקום!

---

## ✅ מה תוקן?

הופכו הערכים המספריים של שני סוגי המסמכים:

### לפני (v1.0.0 - v1.0.2): ❌

```
Payment Request (בקשת תשלום) = 10  ← שגוי!
Return/Credit Note (תעודת החזרה) = 9  ← שגוי!
```

**תוצאה**: בחירה ב-"Payment Request" יצרה "Return/Credit Note"

---

### אחרי (v1.0.3): ✅

```
Payment Request (בקשת תשלום) = 9  ← נכון!
Return/Credit Note (תעודת החזרה) = 10  ← נכון!
```

**תוצאה**: עכשיו זה עובד נכון!

---

## 🔄 איך לעדכן? (חובה!)

### שלב 1: עדכן את החבילה

```bash
npm update n8n-nodes-sumit
# או
npm install n8n-nodes-sumit@latest
```

### שלב 2: הפעל מחדש את n8n

```bash
# עצור את n8n (Ctrl+C)
# נקה cache (מומלץ)
rm -rf ~/.n8n/cache

# הפעל מחדש
n8n start
```

### שלב 3: רענן את הדפדפן

לחץ **Ctrl+Shift+R** (או **Cmd+Shift+R** ב-Mac) כדי לרענן את n8n ולטעון את הנוד המעודכן.

### בדיקה מהירה:

1. **צור workflow חדש**

3. **בחר**:
   - Resource: `Document`

5. **הפעל**
6. **בדוק בסאמיט** - אם נוצרה **בקשת תשלום** ✅ - זה עובד!

## 🤔 למה זה קרה?

הבעיה הייתה שבתיעוד המקורי של Sumit API **לא היו רשומים הערכים המספריים** של כל סוג מסמך בצורה מפורשת.

הנחתי סדר הגיוני, אבל התברר שהערכים של **9 ו-10 הפוכים**.
