## ✅ הפתרון - בדיקה מדויקת

אנחנו צריכים לבדוק **מה באמת נשלח** ו**מה באמת נוצר**.

---

## 🧪 בדיקה 1: מה נשלח ל-API?

### ב-n8n:

1. **פתח את ה-workflow** שיצרת
2. **לחץ על Sumit Node**
3. **בחר את הטאב "Code"** או "Input Data"
4. **חפש את השדה `Type` בתוך `Details`**

**תעתיק לי בדיוק מה הערך של `Type` שנשלח!**

---

## 🧪 בדיקה 2: מה באמת נוצר בסאמיט?

1. **היכנס לסאמיט**
2. **עבור למסמכים**
3. **מצא את מסמך מספר 1000** (זה שנוצר)
4. **לחץ על "פרטים" או "עריכה"**
5. **תראה מה כתוב ב"סוג מסמך"**

---

## 🔍 בדיקה 3: קריאה חזרה מה-API

בואו נשתמש ב-API כדי לקרוא את המסמך שנוצר:

### צור Workflow בדיקה חדש:

```
Manual Trigger
  ↓
HTTP Request Node
    Method: POST
    URL: https://api.sumit.co.il/accounting/documents/getdetails/
    Body:
    {
      "Credentials": {
        "CompanyID": [המספר שלך],
        "APIKey": "[המפתח שלך]"
      },
      "DocumentID": 1264926722
    }
```

**הפעל את זה והעתק לי את התשובה!**

זה יגיד לנו בדיוק מה סוג המסמך שנוצר.
