# 🔍 בדיקת סוגי מסמכים - Debug Guide

## הבעיה

אתה בחרת "Payment Request (בקשת תשלום)" אבל הוא יצר "Return/Credit Note (תעודת החזרה)".

זה אומר שהערכים המספריים לא תואמים למה שסאמיט מצפה!

---

## 🧪 בדיקה: מה הערך שנשלח?

### שלב 1: בדוק מה נשלח בפועל

1. **פתח את ה-workflow שיצר תעודת החזרה**
2. **לחץ על Sumit Node**
3. **לחץ על "Execute Node"**
4. **לחץ על "JSON" בתוצאות**
5. **חפש את השדה `Type` בתוך `Details`**

זה יראה לך מה נשלח בפועל ל-API.

## 🤔 אפשרויות למה קרה

### אפשרות 1: Cache ישן
אולי n8n שמר cache ישן. **פתרון**:
```bash
# עצור את n8n
# נקה cache
rm -rf ~/.n8n/cache

# התקן מחדש את הנוד
npm install n8n-nodes-sumit@latest

# הפעל
n8n start
```

### אפשרות 2: הגדרות של סאמיט שונות
אולי בחשבון סאמיט שלך, סוגי המסמכים מוגדרים אחרת?

**בדיקה**:
1. היכנס לחשבון סאמיט שלך
2. עבור ל-**הגדרות** → **מסמכים**
3. בדוק איזה סוגי מסמכים פעילים

### אפשרות 3: הערכים ב-API של סאמיט שונים
אולי Sumit שינתה את הערכים המספריים?

**פתרון**: נצטרך לבדוק עם התיעוד הרשמי או ה-support של סאמיט.

## 🔬 בדיקה מדויקת - צור Workflow בדיקה

### Workflow לבדיקה:

```
Manual Trigger
  ↓
Function Node (הכן נתונים)
  ↓
HTTP Request Node (שלח ישירות ל-API)
```

### Function Node - הכן נתונים:

```javascript
// נבדוק כל ערך
return [{
  json: {
    testType: 10, // ערך של Payment Request
    endpoint: '/accounting/documents/create/',
    body: {
      Credentials: {
        CompanyID: 12345678, // החלף במספר שלך
        APIKey: 'your-api-key' // החלף במפתח שלך
      },
      Details: {
        Type: 10, // <- זה מה שאנחנו בודקים
        Customer: {
          SearchMode: 1,
          ID: 123 // ID של לקוח קיים
        },
        IsDraft: true // נשמור כטיוטה לבדיקה
      },
      Items: [{
        Item: { Name: "בדיקה", SearchMode: 0 },
        Quantity: 1,
        UnitPrice: 100
      }],
      VATIncluded: true
    }
  }
}];
```

### HTTP Request Node:

```
Method: POST
URL: https://api.sumit.co.il/accounting/documents/create/
Body Type: JSON
Body: {{ $json.body }}
```

אם זה גם יוצר תעודת החזרה - **הבעיה היא ב-API של סאמיט!**

## 💡 פתרון זמני

עד שנברר מה הבעיה, אפשר:

1. **לבדוק בממשק של סאמיט** איזה מסמך נוצר (מספר מסמך, סוג)
2. **לשנות ידנית** בסאמיט את סוג המסמך
3. **להשתמש ב-API ישירות** (דרך HTTP Request) ולבדוק כל ערך
